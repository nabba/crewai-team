# Multi-Step Data Pipeline Orchestration

## Problem
Tasks requiring high-dependency chains (e.g., searching for a lead, validating their API, and generating a report) often suffer from 'context drift' or failed hand-offs between the research, coding, and writing crews.

## Solution: The Pipeline Manifest Pattern
1. **Manifest Creation**: The lead agent must create a `pipeline_manifest.json` in the workspace containing:
   - `objective`: Final goal.
   - `dependencies`: List of required data points.
   - `state`: Current status of each dependency (Pending/Completed/Failed).
   - `artifacts`: Paths to files generated by previous steps.
2. **Crew Hand-off**: Each crew must read the manifest, update the state of their specific contribution, and write the output to the designated artifact path.
3. **Validation**: The self_improvement crew validates the manifest completion before final synthesis.
