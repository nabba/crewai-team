Signal Commands Reference
Send any message to the bot via Signal on your iPhone. The Commander agent automatically classifies your request and routes it to the right specialist crew (Research, Coding, Writing, Media). You can also use the commands below to control the system, manage proposals, monitor evolution, and access diagnostics. File attachments (PDF, DOCX, XLSX, images) are automatically extracted and included as context.
CommandDescription
📱 General — Natural Language Interaction
Send any question or request. Commander classifies it by type and difficulty (1-10), then dispatches to the right crew. Multiple requests can be processed in parallel.
any question or requestCommander routes to Research, Coding, Writing, or Media crew based on content
📎 Send file(s)Attach PDF, DOCX, XLSX, JPG, PNG — text is extracted and included as task context
statusSystem health overview: composite score, pending proposals, LLM mode, error count
llmDetailed LLM configuration: current mode, cost mode, model assignments per role
🤖 LLM Fleet — Model Management
The system manages 18+ models across 5 tiers (local/free/budget/mid/premium). LLM mode controls which tiers are used. Cost mode controls quality vs cost trade-off per role.
mode <local|hybrid|cloud|insane>Switch LLM mode. Local = Ollama only. Hybrid = try local → API → Claude. Cloud = API only. Insane = premium everywhere.
fleetOllama fleet status: loaded models, GPU memory, speeds
catalogFull model catalog with tiers, costs, and current role assignments
tokens [hour|day|week|month]Token usage and cost breakdown by model and time period
fleet pull <model>Download and load a new Ollama model (e.g. fleet pull gemma3:27b)
fleet stop allUnload all models from GPU memory
📚 Learning — Self-Improvement
The system continuously learns new topics and stores knowledge as skill files. Learning runs at 3 AM daily and during idle time. Skills are semantically matched and injected into relevant tasks.
learn <topic>Add a topic to the learning queue (e.g. learn kubernetes networking)
show learning queueView pending topics in the queue
watch <youtube_url>Extract video transcript and distill into a skill file
run self-improvement nowProcess the learning queue immediately (bypasses schedule)
improveScan for skill gaps and create improvement proposals
📋 Proposals — Human-Gated Self-Modification
The system creates proposals for improvements (skills, code fixes, config changes). All code changes require your explicit approval. Approved code is deployed to the live system with hot-reload — no restart needed.
proposalsList all pending proposals with IDs, types, and titles
diff <id>View detailed proposal: description, reasoning, and file contents
approve <id>Approve and deploy a proposal. Code changes are validated (AST + security) and hot-reloaded.
reject <id>Reject a proposal (with reasoning stored for future learning)
rollback <id>Revert a previously approved proposal from backup
🧬 Evolution — Autonomous Self-Improvement
DGM-inspired evolution loop: propose → mutate → evaluate → keep/discard. Each mutation is tested against real tasks. Runs every 6h and during idle time. Max 3 promoted mutations per day. Variant genealogy tracks lineage.
evolveRun an evolution session (5 experiments)
evolve deepExtended evolution session (15 experiments)
experimentsShow recent experiment journal (keep/discard history)
resultsShow results ledger (TSV format, all experiments)
variantsShow variant archive with genealogy (DGM-style lineage)
metricsFull system metrics: composite score, component breakdowns
programView evolution research directions (editable program.md)
🛡️ Self-Healing & Monitoring
Errors are automatically logged, diagnosed, and fixed. The anomaly detector monitors system health metrics and triggers healing when deviations exceed 2σ. Error patterns are tracked with a 24h resolution cooldown.
errorsShow recent errors with type, crew, and diagnosis status
fix errorsManually trigger error resolution loop for recurring patterns
anomaliesShow recent anomaly detection alerts (metric deviations)
auditRun a code quality audit across the codebase
audit statusShow audit activity, error resolution status, and deploy log
deploysShow recent deployment log (auto-deploys from approved proposals)
diagnoseManually trigger error diagnosis for recent undiagnosed errors
auto deployCheck auto-deploy status (enabled/disabled)
auto deploy onEnable auto-deploy: code mutations deploy without approval (with 60s monitoring)
auto deploy offDisable auto-deploy: code proposals require human approval (default)
🔬 Tech Radar — Internet Technology Monitor
Scans the internet for new LLM models, agent frameworks, and research papers. Runs daily during idle time. High-relevance discoveries trigger a Signal notification. New models are auto-proposed for catalog addition.
tech radarShow recent technology discoveries (models, frameworks, papers)
📖 Knowledge Base — Enterprise RAG
Ingest documents (PDF, DOCX, XLSX, URLs) into a semantic search index. All agents automatically search the KB for relevant context before answering questions. Upload via Signal attachment or dashboard.
kbKnowledge base status: document count, chunks, estimated tokens
kb listList all ingested documents with format, category, and chunk count
kb add <url> [category]Ingest a URL into the knowledge base (e.g. kb add https://docs.example.com technical)
📎 + kb add [category]Ingest attached file(s) into the KB (send file with message "kb add research")
kb search <query>Semantic search across all ingested documents
kb remove <source>Remove a document from the knowledge base
kb resetClear the entire knowledge base
🧠 Meta-Cognitive — Introspection
The retrospective crew analyzes past performance and generates TRIGGER/ACTION/EVIDENCE policies. Policies are automatically injected into future tasks to improve behavior. The system runs retrospectives at 4 AM and during idle time.
retrospectiveRun a meta-cognitive retrospective analysis now
benchmarksShow performance benchmarks and period-over-period trends
policiesShow stored improvement policies (TRIGGER → ACTION)
