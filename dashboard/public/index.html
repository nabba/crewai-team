❓ Signal Commands Reference
CommandWhat happens
General
hello / hi / any questionCommander routes to the right crew (research, coding, writing) or answers directly. Short replies use conversation history for context.
📎 Send file(s)Attach PDF, DOCX, XLSX, JPG, or PNG to any message. The system extracts text/data and includes it as context. Works with all crews.
statusReturns system health, pending proposals, composite score, and LLM info.
Learning & Self-Improvement
learn <topic>Adds topic to the learning queue. At 3 AM daily (or manually), the self-improvement crew researches it and saves a skill file.
watch <youtube_url>Extracts video transcript, distills key knowledge into a skill file, and stores a summary in shared team memory.
show learning queueDisplays all pending topics in the learning queue.
run self-improvement nowImmediately processes the learning queue (up to 3 topics). Agents research, extract knowledge, and save skill files.
improveTriggers an improvement scan. An analyst reviews current skills and creates 1-3 structured improvement proposals.
Proposals
proposalsLists all pending improvement proposals with IDs, types, and titles.
approve <id>Approves and applies a proposal. Skill proposals copy files immediately; code proposals are staged for next rebuild.
reject <id>Rejects and archives a proposal. No files are changed.
Evolution & Experiments
evolveTriggers one evolution cycle. The system analyzes metrics, proposes ONE improvement, applies it, and keeps or discards based on score delta.
evolve deepRuns an extended evolution session with more iterations.
experimentsShows the experiment journal: what evolution has tried, kept, and discarded (last 15).
resultsDisplays the results ledger with score history (last 20).
Errors & Auditing
errorsShows the 5 most recent errors with diagnosis status and recurring patterns.
auditTriggers an immediate code audit. The auditor reads all Python source files, identifies bugs/security issues, and auto-deploys fixes.
fix errorsTriggers the error resolution loop. Scans for recurring patterns, generates targeted fixes, applies them, and monitors recurrence (up to 3 attempts).
auditorShows audit activity summary: recent code audits, error resolution status, and recent auto-deployments.
deploysShows the auto-deployment log: what files were deployed, when, and why (last 10).
LLM & Fleet Management
llmShows current LLM configuration: mode, last model used, tier, vetting status, and cost breakdown.
mode <local|hybrid|cloud>Switches LLM mode. mode alone shows current mode.
fleetShows LLM fleet status: running containers, available models, and benchmark scores.
fleet stop allStops all fleet containers immediately, freeing all GPU/RAM.
fleet pull <model>Pulls a new model from Ollama registry. Example: fleet pull gemma3:27b
catalogShows the full LLM model catalog with tiers, costs, and role assignments.
tokensShows token usage for today. Add period: tokens week, tokens month, etc.
Analysis & Monitoring
metricsDisplays system metrics: composite score, task success rate, error rate, self-heal rate, and more.
benchmarksShows model performance benchmarks across task types.
policiesShows all active self-learned policies that guide agent behavior.
retrospectiveRuns a retrospective analysis of recent execution data to discover patterns and generate new policies.
programShows the evolution program (research directions from workspace/program.md).
Knowledge Base
kbShows knowledge base status: document count, chunk count, categories, and estimated tokens.
kb listLists all documents in the knowledge base with format, category, and chunk counts.
kb add <url> [category]Ingests a URL into the knowledge base. Optional category (default: general). Example: kb add https://docs.example.com technical
📎 + kb add [category]Send a file attachment with text "kb add" to ingest it. Supports PDF, DOCX, PPTX, XLSX, CSV, TXT, MD, HTML, JSON.
kb search <query>Searches the knowledge base and returns top matching passages with relevance scores and sources.
kb remove <source>Removes a document from the knowledge base by its source path.
kb resetDeletes all data from the knowledge base. Use with caution.
How routing works
Commander classifies every message and dispatches to specialist crews:
• Research — web search, fact-finding, current events (can spawn parallel sub-agents for complex topics)
• Coding — writes, tests, and debugs code in a Docker sandbox
• Writing — summaries, documentation, emails, reports
• Direct — simple questions answered by Commander itself
Complex requests can dispatch multiple crews in parallel.
LLM tiers & difficulty routing
Commander classifies task difficulty (1-10). Easy tasks (1-3) → cheap local models. Hard tasks (8-10) → premium cloud models. Medium (4-7) → balanced selection.
Vetting → 4-tier risk-based verification: none (premium+easy), schema (format check), cheap (budget model yes/no), full (Claude Sonnet review). Code always gets full vetting.
Self-healing
When any crew fails: logs error with context → spawns background "System Doctor" agent → diagnoses root cause → saves knowledge fix or creates code proposal.
Evolution loop (autoresearch-inspired)
Every 6 hours: gathers metrics → identifies highest-impact improvement → executes ONE change → keeps improvements, discards regressions. ETAs are dynamically calculated from historical task durations.
Circuit breakers
Each LLM provider (Ollama, OpenRouter, Anthropic) has a circuit breaker. After 3 consecutive failures, the provider is skipped for 60s cooldown, preventing slow timeouts.
