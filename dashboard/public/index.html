❓ Signal Commands Reference
CommandWhat happens
hello / hi / any questionCommander routes to the right crew (research, coding, writing) or answers directly. Short replies are interpreted in context of your conversation history.
📎 Send file(s)Attach PDF, DOCX, XLSX, JPG, or PNG to any message. The system extracts text/data from attachments and includes it as context for the request. Works with all crews. Send files alone to get a summary, or with a message like "summarize this PDF".
statusReturns system health and count of pending improvement proposals.
learn <topic>Adds topic to the learning queue. At 3 AM daily (or on manual trigger), the self-improvement crew researches it and saves a skill file to workspace/skills/.
watch <youtube_url>Extracts the video transcript, distills key knowledge into a skill file, and stores a summary in shared team memory. The learned content improves all future agent responses.
show learning queueDisplays all pending topics in the learning queue.
run self-improvement nowImmediately processes the learning queue (up to 3 topics). Agents research, extract knowledge, and save skill files.
improveTriggers an improvement scan. An analyst agent reviews current skills, tools, and crews, identifies capability gaps, and creates 1-3 structured improvement proposals.
proposalsLists all pending improvement proposals with their IDs, types, and titles.
approve <id>Approves and applies a proposal. Skill proposals copy files to skills/ immediately. Code proposals are staged to applied_code/ and activate on next container rebuild.
reject <id>Rejects and archives a proposal. No files are changed.
errorsShows the 5 most recent errors with diagnosis status and recurring patterns. The self-healing system automatically diagnoses errors and creates fixes in the background.
How routing works
Commander classifies every message and dispatches to specialist crews:
• Research — web search, fact-finding, current events (can spawn parallel sub-agents for complex topics)
• Coding — writes, tests, and debugs code in a Docker sandbox
• Writing — summaries, documentation, emails, reports
• Direct — simple questions answered by Commander itself
Complex requests can dispatch multiple crews in parallel (e.g. research + writing).
Self-improvement flow
learn topic → queued → cron or manual trigger → agent researches 3+ sources → saves skill .md file → skill loaded into all future requests
watch youtube_url → extracts transcript → agent distills key knowledge → saves skill file + stores in team memory → improves future responses
improve → analyst scans capabilities → creates proposals → you approve/reject via Signal → approved changes applied
Self-healing
When any crew fails, the system automatically: logs the error with context → spawns a background "System Doctor" agent → diagnoses the root cause → either saves a knowledge fix (immediate) or creates a code fix proposal (needs approval). Use errors to check the error journal and diagnosis status.
